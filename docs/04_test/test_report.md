# 테스트 계획 및 결과 보고서

## 1. 테스트 목적

- 감정 분석 정확도 검증
- 위험도 분류 성능 확인
- 답변 생성 품질 평가
- 시스템 전체 동작 안정성 확인

---

## 2. 테스트 방법

### 2.1 테스트 유형

- 기능 테스트
- 시나리오 기반 테스트
- 출력 검증 테스트

---

### 2.2 테스트 데이터

다양한 갈등 수준의 대화 시나리오 사용:

1. 일상 대화 (안전)
2. 경미한 갈등 (주의)
3. 중간 갈등 (경고)
4. 심각 갈등 (위험/심각)

---

## 3. 테스트 케이스

### 3.1 안전 대화

입력:
- "오늘 뭐 먹을까?"
- "파스타 먹자"

결과:
- 감정: 중립 / 긍정
- 위험도: 안전

---

### 3.2 경미한 갈등

입력:
- "오늘도 늦었네"
- "답답해"

결과:
- 감정: 슬픔 / 분노
- 위험도: 주의

---

### 3.3 중간 갈등

입력:
- "왜 여자 번호가 이렇게 많아?"
- "짜증나네"

결과:
- 감정: 분노
- 위험도: 경고

---

### 3.4 심각 갈등

입력:
- "헤어지자"
- "죽겠다 진짜"

결과:
- 감정: 혐오 / 분노
- 위험도: 심각

---

## 4. 테스트 결과

### 4.1 감정 분석
- 대부분 상황에서 적절한 감정 분류 수행

---

### 4.2 위험도 분석
- 단계별 구분 정상 작동
- 극단적 표현에서 높은 위험도 출력

---

### 4.3 답변 생성
- 3가지 스타일 모두 생성 성공
- 실제 사용 가능한 문장 형태 유지

---

## 5. 문제점 및 개선

### 5.1 문제점

- LLM 출력 형식 불안정
- 일부 경우 답변 누락
- 비연인 데이터 혼입

---

### 5.2 개선 사항

- repair prompt 적용
- 출력 파싱 로직 강화
- 관계 필터링 추가

---

## 6. 결론

본 시스템은 감정 분석, 위험도 판단, 답변 생성 기능이  
전체적으로 안정적으로 동작함을 확인하였다.

다만 LLM 특성상 출력 안정성 개선이 필요하며,  
데이터 확장을 통해 성능 향상이 가능하다.
