# 테스트 계획 및 결과 보고서

## 1. 테스트 목적

- 감정 분석 결과의 적절성 검증
- 갈등 위험도 분류 단계의 정상 동작 확인
- RAG 기반 답변 생성 품질 검증
- 시스템 전체 파이프라인의 안정성 확인

---

## 2. 테스트 방법

### 2-1. 테스트 유형

- 기능 테스트 (감정 / 위험도 / 답변 생성)
- 시나리오 기반 테스트 (갈등 수준별 입력)
- 출력 구조 검증 테스트 (UI 파싱 및 표시)

---

### 2-2. 테스트 데이터

다양한 갈등 수준을 반영한 대화 시나리오를 구성하여 테스트 수행

1. 일상 대화 (안전)
2. 경미한 갈등 (주의)
3. 중간 갈등 (경고)
4. 심각 갈등 (위험/심각)

---

## 3. 테스트 케이스

### 3-1. 안전 대화

입력:
- "오늘 뭐 먹을까?"
- "파스타 먹자"

결과:
- 감정: 중립 / 긍정
- 위험도: 안전

---

### 3-2. 경미한 갈등

입력:
- "오늘도 늦었네"
- "답답해"

결과:
- 감정: 슬픔 / 분노
- 위험도: 주의

---

### 3-3. 중간 갈등

입력:
- "왜 여자 번호가 이렇게 많아?"
- "짜증나네"

결과:
- 감정: 분노
- 위험도: 경고

---

### 3-4. 심각 갈등

입력:
- "헤어지자"
- "죽겠다 진짜"

결과:
- 감정: 혐오 / 분노
- 위험도: 심각

---

### 3-5. 평가 기준

- 감정: 입력 문장의 주요 감정과 일치 여부
- 위험도: 갈등 강도에 따라 단계적으로 상승하는지 여부
- 답변 생성: 공감형 / 조언형 / 갈등 완충형 3종 생성 여부
- 표현 가이드: 피해야 할 표현 / 대체 표현 정상 분리 여부

---

## 4. 테스트 결과

### 4-1. 감정 분석 성능

- 테스트 케이스 기준 전반적으로 기대 감정과 일치하는 결과 확인
- 일부 모호한 표현에서는 중립으로 분류되는 경향 존재

---

### 4-2. 위험도 분석
- 갈등 수준에 따라 단계적으로 위험도가 상승하는 정상 동작 확인
- 극단적 표현(이별, 분노)에서 높은 위험도 출력

---

### 4-3. 답변 생성
- 공감형 / 조언형 / 갈등 완충형 3종 답변 생성 정상 수행
- 실제 메시지로 활용 가능한 문장 형태 유지

---

### 4-4. 출력 구조 안정성

- 파싱 로직을 통해 UI 출력 구조 정상 유지
- 일부 케이스에서 출력 누락 발생 → 파싱 로직 개선으로 보완

---
### 4-5. 실패 및 한계 케이스

- 은유적 표현에서 감정 강도가 과소 평가되는 경우 존재
- 일부 입력에서 답변 스타일이 중복되는 문제 발생 가능
- LLM 출력 형식이 완전히 일정하지 않은 한계 존재

---

### 4-6. RAG 검색 검증

- 입력 문장과 유사한 갈등 사례가 정상적으로 검색됨
- 검색 결과가 답변 생성에 반영되는 구조 확인

한계:
- 입력 표현에 따라 검색 품질 편차 존재
- 키워드 중심 입력에서 검색 정확도 차이 발생

---

### 4-7. 모델 선택 근거

- Gemini 모델을 감정 및 위험도 분석에 활용
- GPT 모델을 답변 생성에 활용

선택 이유:
- Gemini: 감정 분류 및 추론 성능 우수
- GPT: 자연스러운 문장 생성 및 표현력 우수

---

## 5. 문제점 및 개선

### 5-1. 문제점

- LLM 출력 형식의 불안정성
- 일부 케이스에서 답변 누락 발생
- RAG 검색 결과의 일관성 부족

---

### 5-2. 개선 사항

- 출력 파싱 로직 강화
- 불필요한 표현 필터링 로직 추가
- repair 로직 비활성화 후 구조 기반 안정화 적용
- RAG 검색 품질 개선 필요

---

## 6. 결론

본 시스템은 감정 분석, 위험도 판단, RAG 기반 검색, 답변 생성 기능이  
전체적으로 정상 동작함을 확인하였다.

다만 LLM 특성상 출력 구조의 불안정성과  
RAG 검색 결과 편차가 존재하여  
출력 안정화 및 검색 품질 개선이 향후 과제로 남는다.
