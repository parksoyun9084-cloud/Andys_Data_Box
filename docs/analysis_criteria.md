# 감정 라벨 및 갈등 위험도 기준표

---

## 1. 문서 목적

본 문서는 **연인 갈등 상황 대응 감정 기반 RAG 챗봇**에서 사용되는  
감정 분류 기준과 갈등 위험도 산출 기준을 정의한다.

해당 기준은 다음 모듈에서 공통으로 사용된다.

- `emotion_analyzer.py`
- `risk_analyzer.py`

---

## 2. 감정 분류 기준

### 2.1 감정 라벨 (7종)

본 프로젝트는 데이터셋 기반으로 다음 7개의 감정 라벨을 사용한다.

| 라벨 | 영문 | 설명 | 데이터 분포 |
|---|---|---|---|
| 중립 | neutral | 감정 표현이 없는 일반 대화 | 78.7% |
| 놀람 | surprise | 예상치 못한 상황 반응 | 8.8% |
| 분노 | anger | 화남, 공격적 표현 | 6.5% |
| 슬픔 | sadness | 서운함, 우울 | 3.5% |
| 행복 | happiness | 긍정적 감정 | 1.9% |
| 혐오 | disgust | 경멸, 거부감 | 0.4% |
| 공포 | fear | 불안, 두려움 | 0.2% |

---

### 2.2 감정 상위 그룹 (3단계)

챗봇 응답 전략 수립을 위해 감정을 3개 그룹으로 재분류한다.

| 그룹 | 포함 감정 | 의미 | 대응 전략 |
|---|---|---|---|
| negative | 분노, 슬픔, 혐오, 공포 | 갈등 가능성 높음 | 공감 + 진정 유도 |
| neutral | 중립 | 감정 없음 | 상황 파악 질문 |
| positive | 행복, 놀람 | 긍정 상태 | 긍정 강화 |

> 놀람은 상황에 따라 달라질 수 있으나, 본 프로젝트에서는 긍정으로 처리한다.

---

### 2.3 감정 기반 지표 (Feature Engineering)

위험도 판단을 위해 다음 지표를 계산한다.

| 지표 | 설명 |
|---|---|
| negative_ratio | 부정 감정 비율 |
| anger_ratio | 분노/혐오 집중도 |
| emotion_volatility | 감정 변화 빈도 |
| negative_streak | 연속 부정 감정 길이 |
| ending_sentiment | 마지막 감정 |

---

## 3. 갈등 위험도 정의

### 3.1 위험도 등급 (5단계)

| 레벨 | 라벨 | 설명 | 대응 전략 |
|---|---|---|---|
| 1 | safe | 갈등 없음 | 일반 응답 |
| 2 | caution | 경미한 불만 | 공감 + 질문 |
| 3 | warning | 갈등 시작 | 적극 공감 |
| 4 | danger | 강한 갈등 | 감정 완화 |
| 5 | critical | 관계 단절 위험 | 상담 유도 |

---

### 3.2 위험도 산출 방식

다음 5개 시그널의 가중 합으로 계산한다.

| 요소 | 가중치 |
|---|---|
| negative_ratio | 0.25 |
| anger_ratio | 0.20 |
| keyword_score | 0.25 |
| streak_score | 0.15 |
| ending_score | 0.15 |

> 최종 점수는 0~1 범위로 정규화한다.

---

## 4. 위험 키워드 체계

위험도 판단 정확도를 높이기 위해 키워드를 5단계로 구분한다.

### 4.1 극위험 (critical, 1.0)
- 죽고싶다, 같이죽자, 때리겠다, 칼 등

### 4.2 고위험 (high, 0.8)
- 헤어지자, 끝내자, 꺼져, 연락하지마 등

### 4.3 중위험 (medium, 0.6)
- 한심하다, 쓰레기, 무능, 배신 등

### 4.4 주의 (elevated, 0.4)
- 짜증난다, 화난다, 믿을수없다 등

### 4.5 저위험 (low, 0.2)
- 서운하다, 속상하다, 답답하다 등

---

## 5. 분류 방식 (핵심 설계)

본 프로젝트는 **Rule-Based + LLM 하이브리드 구조**를 사용한다.

### 5.1 Rule-Based

- 키워드 + 통계 기반
- 빠르고 안정적
- 실시간 처리용

### 5.2 LLM 기반

- 문맥 이해 가능
- 감정 복합 판단 가능
- 비용/속도 부담 존재

---

### 5.3 결합 전략

1. Rule-Based 1차 분석 수행
2. 경계 상황에서 LLM 재분석
3. 결과 불일치 시 재검증

## 6. 출력 구조

### 감정 분석 결과

```json
{
  "emotion": {
    "primary": "분노",
    "group": "negative",
    "confidence": 0.85
  }
}
```

### 위험도 분석 결과

```json
{
  "risk_score": 0.72,
  "risk_level": "danger",
  "signals": {
    "negative_ratio": 0.65,
    "keyword_score": 0.80
  }
}
```

## 7. 설계 특징

본 기준은 다음 특징을 가진다.

- 데이터 기반 감정 분류 (7종 → 3그룹)
- 정량적 위험도 계산 구조
- 키워드 기반 해석 가능 모델
- LLM 결합으로 문맥 보완
- 실시간 처리와 정확도 균형 설계
