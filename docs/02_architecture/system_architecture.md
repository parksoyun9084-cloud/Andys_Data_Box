# 시스템 아키텍처 설계서

---

## 1. 개요

본 시스템은 연인 관계 갈등 상황에서 사용자의 입력을 분석하여  
감정 상태와 갈등 위험도를 판단하고,  
실제 메시지로 바로 사용할 수 있는 답장을 추천하는 AI 기반 서비스이다.

본 시스템은 다음 2가지 목표를 중심으로 설계되었다.

- **정확한 상황 이해 (감정 + 위험도 분석)**
- **실용적인 답변 생성 (RAG 기반 추천)**

이를 위해 **LLM + RAG 혼합 구조**를 채택하였다.

---

## 2. 전체 아키텍처

### 2-1. 처리 흐름

```
사용자 입력
→ 갈등 유형 선택 + 텍스트 입력
→ 감정/위험도 분석 (Gemini)
→ RAG 검색 (BM25 + Dense + RRF)
→ 답변 생성 (GPT)
→ 출력 파싱 및 보정
→ Streamlit UI 출력
```

### 2-2. 아키텍처 설계 의도

| 설계 요소 | 이유 |
|---|---|
| Gemini 사용 | 감정 및 문맥 이해 정확도 |
| RAG 도입 | hallucination 방지 + 실제 사례 기반 응답 |
| BM25 + Dense | 키워드 + 의미 검색 결합 |
| RRF | 검색 결과 안정성 확보 |
| GPT 답변 생성 | 자연스러운 한국어 답변 생성 |

---

## 3. 시스템 구성 요소

### 3-1. Frontend (Streamlit)

**역할**
- 사용자 입력 수집
- 분석 결과 시각화
- 채팅형 인터페이스 제공

**특징**
- 빠른 프로토타이핑
- 상태 관리 (`session_state`) 기반 UI

### 3-2. 감정 및 위험도 분석 (Gemini)

**입력**
- 사용자 발화 리스트

**출력**
- 감정 라벨 (7종)
- 감정 흐름
- 갈등 위험도 (5단계)
- 상황 요약
- 피해야 할 표현 / 대체 표현

**설계 이유**
- Rule-based 한계를 보완하기 위해 LLM 사용
- 문맥 기반 감정 해석 가능

### 3-3. RAG 기반 검색 시스템

#### (1) BM25 검색
- 키워드 기반 빠른 검색
- 상황 단어 매칭에 강점

#### (2) Dense 검색
- Embedding 기반 의미 검색
- OpenAI Embedding 사용
- Pinecone 벡터 DB 활용

#### (3) RRF (Reciprocal Rank Fusion)
- BM25 + Dense 결과 통합
- 검색 안정성 및 정확도 향상

### 3-4. 답변 생성 (GPT)

**역할**
- 검색된 문서를 기반으로 자연어 답변 생성

**출력 형태**
- 공감형
- 조언형
- 갈등 완충형

**설계 이유**
- 단일 답변이 아닌 다양한 선택지 제공
- 사용자 상황에 맞는 대응 선택 가능

### 3-5. 출력 보정 및 파싱

**역할**
- LLM 출력 구조화
- 누락된 값 보완
- UI 표시용 데이터 생성

**기법**
- 태그 기반 파싱
- repair prompt 적용

### 3-6. 출력 파싱 상세 로직

LLM 응답은 항상 동일한 형식을 보장하지 않기 때문에  
태그 기반 파싱 및 후처리 로직을 통해 구조를 안정화하였다.

#### 주요 처리 과정

- [상황 요약], [감정], [위험도] 등 태그 기반 분리
- 공감형 / 조언형 / 갈등 완충형 답변 추출
- 피해야 할 표현 / 대체 표현 분리

#### 추가 보정 로직

- 불필요한 문구 제거 (예: "없음", "예:", 특수문자)
- 복수 문장 분리 (쉼표, 줄바꿈 기준)
- UI 표시용 최대 개수 제한 (예: 3개)
- 유효하지 않은 표현 필터링 (`is_valid_phrase`)

👉 이를 통해 LLM 출력의 불안정성을 UI 수준에서 보완하였다.

## 6. 시스템 특징

### 6-1. LLM + RAG 혼합 구조
- 단순 생성이 아닌 검색 기반 생성
- hallucination 감소

### 6-2. 감정 + 위험도 통합 분석
- 감정 상태 + 관계 위험 동시에 고려

### 6-3. 실사용 중심 답변
- 상담형이 아닌 실제 메시지 형태

### 6-4. 출력 안정화
- LLM 불확실성 보완

### 6-5. 핵심 차별점 (Why This System)

본 시스템은 기존 챗봇과 달리 다음과 같은 차별점을 가진다.

- 단순 상담형 응답이 아닌 "실제 메시지 형태 답장 생성"
- 감정 + 위험도를 동시에 고려한 대응 전략 제공
- RAG 기반 실제 연인 갈등 사례 활용 (환각 감소)
- LLM 출력 구조 파싱 및 후처리 기반 안정화
- 사용자 입력 기반 상황 유지 (임의 상황 생성 방지)

👉 단순 정보 제공이 아닌 "행동 가능한 답변 생성"에 초점을 둔다.

## 9. 설계 의도 

본 시스템은 다음을 목표로 한다.

- 사용자가 **즉시 사용할 수 있는 답장 제공**
- 감정 + 상황 + 위험도를 동시에 고려
- 실제 관계 문제 해결에 도움을 주는 AI

### 9-1. 실제 서비스 관점 설계

본 시스템은 단순한 모델 조합이 아니라  
"사용자 입력 → 분석 → 검색 → 생성 → 파싱 → UI 출력"까지  
전체 파이프라인이 연결된 구조로 구현되었다.

👉 각 단계는 독립 모듈로 구성되어 유지보수 및 확장이 가능하도록 설계하였다.

---

## 10. 결론

본 아키텍처는 단순 챗봇이 아닌  
**상황 이해 + 감정 분석 + 실제 행동 추천**을 결합한 구조이다.

이를 통해 기존 LLM 기반 서비스의 한계를 보완하고  
실제 사용자에게 실질적인 가치를 제공하는 것을 목표로 한다.
